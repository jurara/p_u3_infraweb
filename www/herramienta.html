Herramientas y técnicas para incrementar la calidad en el código de tu proyecto.

conferencista:

Ing. Eduardo Figarola Mota

El Software perfecto no existe, solo existe software de gran calidad.

Lo primordial para que los desarrolladores desarrollen buen código es mantenerlos contentos con lo que ellos quieran comer y de paso una michelada, para esto se contrata un becario el cual es el que va por la despensa de la semana. También se tienen que mantener relajados para lo cual se recomienda contratar una masajista la cual les de masaje una vez a la semana.

  • Para que un Software sea de calidad hay 8 puntos que debe de cumplir:
    • primero tiene que cumplir con su propósito.
    • Informar adecuadamente cuando ocurre un problema.
    • Tiene pruebas
    • Se entiende su intención con solo leerlo
    • Sigue un estilo de código desde el inicio
    • Utiliza un flujo de colaboración adecuado
    • Implementa correctamente los patrones de diseño
    • Se puede utilizar intuitivamente
  • Sistemas detectores de errores:
    • Bugsnag
    • Sentry
    • Honey badger
    • Airbreak.io
    • Guitest
    • Integration
  • Legibilidad y estilo:
    • Es lint

Una práctica muy importante es hacer un buen control de versiones para ello existen unas herramientas muy buenas llamadas git y github las cuales trabajan juntas haciendo un control de versiones del programa que se esté desarrollando. En este programa puedes hacer varias ramas en el cual llevar el control de versiones, para esto se recomienda hacer las siguientes ramas mas las que crea necesarias. Una rama estable para producción en la cual se encuentre el Software ya probado y sin fallas. Una rama para el ambiente de producción en la cual se encuentre el Software a modificar. Y definir un estándar para los nombres de las ramas de trabajo individual.

También se recomienda hacer un estándar para los mensajes de los commits así como también para como escribir el código o nombrar a las variables esto hará que sea más fácil revisar el código.
